# ADR-0009 — Scheduler & Provider Catalog

Status: Accepted

## Context

La Alpha dejó dos piezas listas para la Beta:

- `kaos run` resume las suscripciones activas y **publica solo si hubo cambios**
  (ADR-0008). Faltaba el disparador periódico que convierte a KAOS en un sistema
  que se mantiene solo.
- La selección de proveedor LLM (`KAOS_LLM_PROVIDER` + credenciales) vivía únicamente
  en el entorno, sin forma de inspeccionar qué proveedores existen, cuál está
  activo y si tiene credencial. Al sumar `github` y `anthropic` a `openai`/`echo`,
  la pregunta "¿con qué voy a correr?" dejó de ser trivial.

## Decision

### Scheduler

- `src/kaos/runtime/scheduler.py` define `Scheduler`, un lazo **Core-agnóstico**:
  recibe un job async que devuelve un exit code y un `interval`; no conoce Discord,
  el LLM ni Storage. El tiempo es inyectable (`sleep`), por lo que se testea sin
  esperas reales. Duerme `interval` segundos **entre** corridas (nunca después de
  la última), y admite `max_runs` para una pasada única.
- `kaos schedule [--interval N] [--once] [--dry-run] [--consolidated] [--force]`
  envuelve `kaos run`. Como `run` es idempotente, una corrida programada sobre
  foros sin cambios no publica nada y reutiliza la cache (ADR-0007) sin llamar al
  LLM. `--once` hace una sola pasada (para un cron externo); sin `--once` corre
  hasta `Ctrl-C`.
- Intervalo por defecto configurable con `KAOS_SCHEDULER_INTERVAL` (900s).

### Catálogo de providers

- `src/kaos/core/providers.py` describe cada proveedor (`id`, modelo por defecto,
  endpoint, variables de credencial, notas) como **dato puro**, sin importar el
  plugin: el Core no depende de una implementación concreta. Una aserción mantiene
  el catálogo sincronizado con `config.LLM_PROVIDERS`.
- `is_ready(provider_id, settings)` refleja la misma lógica de credenciales que
  `build_llm`, y `provider_status(settings)` devuelve `(info, ready, active)`.
- `kaos providers` lista el catálogo, marca el proveedor activo y si tiene la
  credencial presente. **Nunca imprime secretos**, solo si están configurados.

## Consequences

- KAOS pasa de "lo corro yo" a "se mantiene solo": el Scheduler es el disparador
  natural de `kaos run`, y la regla idempotente evita publicaciones redundantes.
- El Scheduler no depende de ningún plugin, así que a futuro puede orquestar
  cualquier job async (no solo suscripciones de Discord).
- La composición sigue centralizada: `build_llm` es el único lugar que instancia
  proveedores; el catálogo solo los **describe**.
- La selección de modelo (p. ej. Claude Haiku vía Anthropic, o gpt-5 vía GitHub
  Models) es ahora inspeccionable antes de correr, reduciendo errores de config.
