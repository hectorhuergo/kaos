# ADR-0022 — Per-run Provider & Model Selection

Status: Accepted

## Context

Hasta ahora KAOS resolvía **un solo** proveedor + modelo global (el persistido en
`RuntimeConfig`, con `.env` como fallback) para todas las corridas: scheduler,
`kaos run` y consola. En la beta necesitábamos elegir **por corrida** el proveedor
y el modelo en los distintos puntos donde se produce conocimiento:

- **Subscriptions**: la elección debía ser *persistente* para que el scheduler y
  las corridas headless la respetaran, no solo la consola.
- **Chat**: una corrida puntual (override transitorio, no se guarda en la sesión).
- **Vista previa** y **Dashboards**: elegir/editar antes de previsualizar o ejecutar.

Además queríamos poblar un **selector** con los modelos que ofrece cada proveedor,
degradando a un campo de texto libre cuando no se pueden listar.

## Decision

### A. Override en `load_settings`

`load_settings(base, *, provider=None, model=None)` acepta un override explícito
que **gana** sobre la config persistida y determina qué credencial se resuelve.
Un `None` significa "usar la selección global". Para evitar fugas, si el override
cambia de proveedor y no trae modelo, el modelo persistido (de otro proveedor) se
descarta. El override se enhebra por toda la tubería:
`run_backfill`, `run_forum_backfill`, `run_github`, `send_message`,
`preview_subscription` y `run_subscription` (estos dos últimos tomando por defecto
los valores guardados en la suscripción).

### B. Persistencia en la suscripción

`Subscription` gana `llm_provider` / `llm_model` (nullable). PostgreSQL agrega las
columnas con `ADD COLUMN IF NOT EXISTS` (compatibilidad hacia atrás); el store
in-memory ya es genérico. Vacío = usar el default global.

### C. Listado de modelos (best-effort)

`OpenAICompatibleLLMProvider.list_models()` consulta `GET {base}/models` y parsea
la forma OpenAI (`{"data":[{"id":…}]}`) con fallbacks. `factory.list_models(base,
provider_id)` resuelve la credencial del proveedor pedido (aunque no sea el
activo), construye el provider y lista; **cualquier** fallo degrada a `[]`. Nuevo
endpoint `GET /api/providers/{id}/models`.

### D. API + consola

Los bodies de run/preview/chat/subscription aceptan `llm_provider`/`llm_model`
opcionales; los endpoints de suscripción los persisten y devuelven. La consola
suma un helper reutilizable de selector proveedor+modelo (`<select>` cuando hay
modelos, si no un `<input>` de texto) usado en Subscriptions, Chat ("Más
opciones"), Vista previa y un **modal** propio de Dashboards que reemplaza al
`confirm()` y permite editar proveedor/modelo y el toggle Publicar antes de
ejecutar.

## Consequences

- El scheduler y `kaos run` respetan el proveedor/modelo por suscripción sin
  redeploy; el chat puede probar otro modelo sin cambiar el global.
- El selector es best-effort: si un proveedor no expone `/models` (o no hay
  credencial), la UI cae al campo de texto — sin errores.
- Cambio aditivo y compatible: sin override, el comportamiento previo se conserva.
