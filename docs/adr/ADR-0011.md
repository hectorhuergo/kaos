# ADR-0011 — Live Dashboard (FastAPI)

Status: Accepted

## Context

ADR-0010 introdujo el Knowledge Graph y un **dashboard estático** (HTML
exportado con `kaos dashboard`). Sirve para compartir un snapshot, pero no
refleja los cambios sin regenerar el archivo. Para "probar con casos reales" hace
falta una vista **siempre actual** del conocimiento, y una API para integrarlo
con otras herramientas (futuro dashboard rico, front propio, etc.).

`fastapi` ya era dependencia del proyecto; faltaba materializar el servicio.

## Decision

### App FastAPI de solo lectura

- `src/kaos/plugins/dashboard/app.py` define `create_app(settings)` que arma una
  app FastAPI **read-only** (expone conocimiento, nunca lo muta). FastAPI se
  importa de forma **perezosa** dentro de `create_app`, así importar el paquete
  no exige el stack web.
- En cada request lee `Storage` (vía el servicio compartido), de modo que la
  vista siempre está al día sin regenerar archivos.
- Rutas: `GET /` (HTML, mismo renderer que el export), `GET /api/workspaces`,
  `GET /api/knowledge` (grafo JSON) y `GET /api/artifacts`.

### Servicio compartido

- `src/kaos/plugins/dashboard/service.py` centraliza la resolución de workspaces
  (explícito o suscripciones activas) y la carga del grafo + artifacts. Lo usan
  tanto la CLI (`kaos knowledge/dashboard`) como la app, sin duplicar la lógica
  de proyección.

### CLI y dependencias

- `kaos serve [--host 127.0.0.1] [--port 8000]` levanta la app con uvicorn.
- `uvicorn` es un extra opcional (`.[dashboard]`); `kaos serve` lo importa de
  forma perezosa y da un mensaje claro si falta.

## Consequences

- KAOS ofrece dos vistas complementarias: el **export estático** (compartir,
  offline) y el **dashboard vivo** (siempre actual, con API JSON).
- La API abre la puerta a integraciones y a un dashboard más rico (filtros,
  búsqueda, autenticación) sin tocar el Core.
- La app es agnóstica del backend (in-memory o PostgreSQL) y del proveedor LLM:
  solo proyecta artifacts ya persistidos.
- Se mantiene la separación de capas: el modelo del grafo vive en `core`, la
  presentación (HTML/HTTP) en `plugins/dashboard`.
