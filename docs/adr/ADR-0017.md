# ADR-0017 — Console: Agent Catalog, Prompt Augmentation & Interactive Run

Status: Accepted

## Context

La consola web (ADR-0013) exponía providers, suscripciones y una *vista previa*
(dry-run que no persiste ni publica). Faltaban tres cosas que pedía el uso real:

1. **Ver los agentes disponibles** y poder **aumentar su prompt** desde la UI.
2. Un **run interactivo** que genere y **persista** los resúmenes y **llene la
   cache** (no solo previsualizar), con opción de **publicar**.
3. Evitar que la consola dependa de importar las implementaciones de los agentes.

## Decision

### Catálogo de agentes (dato, no wiring)

- `kaos.core.agents`: `AgentInfo` + `AGENT_CATALOG` (id, label, description,
  `produces`, `trigger`, `augmentable`). Es **solo metadata**, análogo al catálogo
  de providers; no registra ni compone agentes (eso sigue en el factory).
- `GET /api/agents` expone el catálogo. La pestaña "Agentes" lo lista.

### Prompt aumentable por agente

- `ResumeAgent(llm, extra_instructions="")` **agrega** las instrucciones al system
  prompt base (no lo reemplaza), preservando la estructura del informe.
- `extra_instructions` se hila por los pipelines (`run_backfill`,
  `run_forum_backfill`, `run_github`) y por las rutas de preview/run.
- En la UI, cada agente **aumentable** tiene su propio campo de prompt extra
  (por id). Hoy el pipeline de resumen consume el del `resume-agent`; el `augmentable`
  del catálogo declara la capacidad y deja lugar a que otros agentes (p. ej.
  `dev-agent`) consuman el suyo cuando se ejecuten desde la consola.

### Run interactivo (persiste + cachea; publica opcional)

- `kaos.plugins.dashboard.execute`: `run_subscription(...)` y `run_all(...)`
  corren el pipeline real con **storage real** (persisten artifacts + **llenan la
  SummaryCache**). Por defecto usan `CapturingPublisher` (no publican); con
  `publish=True` usan un `_TeePublisher` que **publica en Discord y captura** para
  mostrar el resultado en la misma respuesta.
- Rutas: `POST /api/run/subscription` y `POST /api/run/all`
  (`{force?, publish?, extra_instructions?}`). La consola agrega botones
  "Ejecutar", "Ejecutar todas" y checkboxes *force* / *publicar*.
- La **vista previa** sigue siendo dry-run puro (no persiste, no publica).

## Consequences

- La consola pasa de "solo mirar" a **operar**: seleccionar modelo → ajustar el
  prompt del agente → ejecutar (persistir + cache) → opcionalmente publicar →
  verlo en el dashboard/grafo.
- Publicar sigue siendo una acción **deliberada** (checkbox), evitando spamear
  Discord por accidente.
- El catálogo de agentes desacopla la UI de las implementaciones (contratos
  públicos, sin importar plugins), y prepara el terreno para prompts por agente
  más allá del Resume Agent.
