# ADR-0003 — Kernel Contracts

Status: Accepted

## Context

El principio "Toda funcionalidad nueva debe implementarse mediante contratos
públicos" exige materializar el Kernel alrededor de un conjunto estable de
contratos (ver `AGENTS.md` → Initial Contracts). Estos contratos son la
frontera entre el Core y sus plugins.

## Decision

Los contratos se definen en `src/kaos/contracts/` con dos estilos:

- **Estructuras de datos inmutables** con Pydantic v2 (`model_config =
  ConfigDict(frozen=True)`): `Event`, `Context`, `Artifact`.
  - Soportan *Immutable Evidence* y *Everything is Traceable*
    (`Artifact.source_events`).
- **Contratos de comportamiento** con `typing.Protocol` + `runtime_checkable`:
  `Agent`, `Connector`, `Publisher`, `Storage`, `LLMProvider`, `EventBus`,
  `Runtime`.
  - Se usa tipado estructural para favorecer composición sobre herencia; los
    plugins no heredan del Core, simplemente cumplen la forma del Protocol.

El Core permanece agnóstico: los Protocols no importan proveedores externos.
El paquete se marca como tipado con `src/kaos/py.typed`.

## Consequences

- Los plugins se implementan satisfaciendo Protocols, sin acoplarse al Runtime.
- Los contratos deben evolucionar con mucho cuidado y mantener compatibilidad
  hacia atrás siempre que sea posible.
- Cualquier cambio de firma en un contrato es una decisión arquitectónica y
  requiere revisar este ADR.
