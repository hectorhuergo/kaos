# ADR-0006 — Configuration & Composition Root

Status: Accepted

## Context

`kaos up` cableaba un pipeline demo fijo. Para usar plugins reales (Discord,
GitHub Models) sin acoplar el Core a ellos, KAOS necesita configuración externa
y un único lugar que ensamble los plugins.

## Decision

### Configuración

- `src/kaos/core/config.py` define `Settings` (Pydantic), cargado desde el
  entorno con `Settings.from_env()` y el prefijo `KAOS_*`. Es **dato puro**: no
  importa plugins, por lo que el Core sigue agnóstico.
- Variables: `KAOS_LLM_PROVIDER` (echo|openai|github), `KAOS_LLM_MODEL`,
  `KAOS_LLM_API_KEY`, `KAOS_LLM_BASE_URL`, `KAOS_GITHUB_TOKEN`/`GITHUB_TOKEN`,
  `KAOS_DISCORD_TOKEN`, `KAOS_DISCORD_GUILD_ID`, `KAOS_DISCORD_CHANNEL_IDS`,
  `KAOS_RESUME_THREAD_NAME`. Ver `.env.example`.

### Raíz de composición

- `src/kaos/bootstrap/factory.py` (`build_runtime`) es el **único** lugar que
  conoce todos los plugins. Selecciona LLM, connector y publisher según la
  configuración.
- Sin `KAOS_DISCORD_TOKEN`, `kaos up` corre el demo offline (conversación
  estática + consola). Con token, usa el gateway real de Discord y publica en un
  thread de Discord.

### Plugins reales

- `DiscordGatewaySource` (`plugins/connectors/`): fuente en vivo sobre
  `discord.py` (import perezoso; extra opcional `.[discord]`).
- `DiscordPublisher` + `DiscordRestPoster` (`plugins/publishers/`): publica el
  resumen en el thread "📋 Resume" vía la API REST de Discord (httpx).
- `DiscordWebhookPublisher` (`plugins/publishers/`): publica vía webhook (solo
  escritura, sin bot token) en un thread concreto (`?thread_id=`).
- `DiscordBackfillSource` (`plugins/connectors/`): fuente acotada que lee el
  historial de un hilo/canal por REST (paginado, orden cronológico) y lo resume
  una vez. Encaja con el trigger `conversation.completed` sin necesitar un
  Scheduler.

### Persistencia

- `PostgresStorage` (`plugins/storage/`, asyncpg, extra opcional `.[postgres]`):
  persiste eventos y artifacts de forma inmutable en PostgreSQL (JSONB), cableado
  vía `KAOS_DATABASE_URL`. Sin esa variable, se usa `InMemoryStorage`.
  Verificado contra el contenedor de `docker/docker-compose.yml`.

## Consequences

- El Core y los plugins permanecen desacoplados; solo la raíz de composición
  los une.
- La summarización en tiempo real de un gateway en vivo requiere un Scheduler
  que dispare `conversation.completed`; queda pendiente para la Beta (hoy el
  trigger se emite al drenar fuentes acotadas / backfill).
- Se añade `discord.py` como dependencia **opcional**.
